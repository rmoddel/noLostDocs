# Feature Research: Doc Wallet

## Table Stakes

- Fast scan/import and reliable in-app viewing
- Simple wallet-style categorization with large, obvious sections
- App unlock with PIN and biometrics
- Encrypted local storage
- Clear local-only versus cloud-backed positioning
- Restore flow for cloud-backed users
- Device list and remote lock for lost-phone recovery

## Differentiators

- Family or emergency sharing
- Professional templates and compliance-focused document sets
- Expiration reminders and proactive status indicators
- OCR-powered search
- Emergency access workflows

## Anti-Features

- Marketing the app as a legal replacement for government IDs or other originals
- Hiding the local-only data loss tradeoff
- Shipping advanced automation before the secure storage model is proven
- Treating cloud mode as always-on instead of explicit opt-in

## Complexity Notes

- OCR and sharing create the largest privacy/compliance expansion.
- Remote lock is only meaningful when device registration and lock checks are enforced consistently.
- Professional verticals are attractive expansion paths but should not dictate the MVP architecture.
