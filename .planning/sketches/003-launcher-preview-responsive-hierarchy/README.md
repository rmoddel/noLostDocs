# Sketch 003: Launcher Preview Responsive Hierarchy

## Design Question
How should the refined Launcher Preview layout transition from mobile to desktop so the category buttons remain the dominant navigation layer, document buttons remain clearly secondary, and the overall grid feels more intuitive at wider widths?

## How to View
open .planning/sketches/003-launcher-preview-responsive-hierarchy/index.html

## Variants
- **A: Structured Grid** — conservative responsive grid that expands cleanly from 2 columns on mobile to 4 on desktop. ★ Selected
- **B: Featured Primary Category** — one hero category plus supporting categories to create stronger navigation hierarchy.
- **C: Desktop Utility Rail** — category grid stays central while desktop introduces a support rail for trust/auth/secondary actions.

## What to Look For
- Whether categories feel clearly more important than document buttons.
- Whether the mobile-to-desktop transition feels designed instead of merely stretched.
- Which desktop composition keeps the app preview feeling simple and intuitive.
