# Plan 07-01

## Goal

Make OCR the first product-quality focus after cutover by replacing the current basic scan flow with a capture experience that gives the user strong live feedback and fast text extraction.

## Working Recommendation

- use `Scanbot SDK` as the capture layer because it supports mobile apps or websites, user guidance, automatic capture, auto-cropping, and on-device processing
- pair capture with a dedicated OCR extraction pass, likely `ABBYY FineReader`-class OCR for accuracy and layout fidelity, or a later backend OCR service decision if we want a cloud tradeoff
- keep Android-only ML Kit Document Scanner as a secondary reference, not the primary path, because the current product is web-first

## Phase Deliverables

- a polished scan flow with live capture feedback
- clearer OCR result states for valid, blurry, distant, tilted, or invalid documents
- extraction and review states that are better than the current file-upload flow
- a launch note explaining exactly how OCR feeds the premium document workflow
