# Plan 05-01

## Objective

Verify that the Next app has reached cutover-ready parity for the currently scoped MVP surface.

## Tasks

1. Re-run `apps/web-next` typecheck and production build.
2. Verify the public routes, metadata routes, and runtime assets served by the Next app.
3. Verify homepage copy, login, dashboard, scan, contact, and protected-flow surfaces against the current rearchitecture target.
4. Audit the browser/server Supabase boundary and grep for secret-exposure risks in the Next app and shared browser helpers.
5. Record the exact parity status, residual risks, and cutover recommendation.

## Success Criteria

- Next build and typecheck pass.
- Homepage, auth, dashboard, scan, contact, and metadata routes all exist and serve the expected content.
- Runtime assets resolve from `apps/web-next/public`.
- Browser code exposes only publishable Supabase credentials and does not rely on service-role secrets.
- The Vite prototype still exists as the untouched fallback/reference app.
