# Plan 05-02

## Goal

Clear the Phase 5 blockers that prevented cutover by moving protected-route gating to the server boundary, removing protected URLs from the public sitemap, and rerunning verification with real local Supabase env values present for `apps/web-next`.

## Scope

- add cookie-aware Supabase SSR scaffolding in `apps/web-next`
- gate `/dashboard` and `/scan` on the server instead of the client shell
- keep existing product flows and backend functions untouched
- verify with actual `NEXT_PUBLIC_SUPABASE_*` values wired locally for the Next app

## Out of Scope

- cutover or archive work
- changing Supabase migrations or Edge Functions
- redesigning dashboard, scan, or login UX
- replacing the current email auth product flow beyond what is required for SSR-safe session handling

## Execution Notes

1. Install the supported SSR helper package and switch the Next app clients to cookie-aware helpers.
2. Add a server helper for protected pages and a proxy for auth cookie refresh.
3. Remove protected routes from the sitemap and rerun the Phase 5 checks.
4. Update the Phase 5 summary and roadmap/state only if the evidence changes the cutover status.
