# Plan 04-01

## Objective

Port the sensitive account flows into the Next app without touching the Supabase backend or the legacy Vite prototype.

## Tasks

1. Extract document access-state, protected download, and audit helpers into `lib/documents`.
2. Extract device loading, registration, and lock/unlock behavior into `lib/devices` and surface it through reusable dashboard panels.
3. Port the scan flow into dedicated scan components with file validation, rotation, signed upload, and document row creation.
4. Port the contact form into a reusable component with validation and signed-in prefill.
5. Keep plan boundaries explicit in the dashboard UI and verify the Next build after the port.

## Success Criteria

- Protected document actions in the Next dashboard preserve the prototype access rules.
- Device registration and lock/unlock flows call the existing Edge Functions.
- The scan route supports image selection, rotation, validation, and signed upload when configured.
- The contact route submits through `contact-submit` and shows clear success/error states.
- The Next build passes after the protected-flow port.
