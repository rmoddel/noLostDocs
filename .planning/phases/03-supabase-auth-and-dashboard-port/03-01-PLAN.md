# Plan 03-01

## Objective

Port the Phase 3 auth shell and category-first dashboard structure into the Next app.

## Tasks

1. Add a client auth provider and shared login/sign-out helpers using browser-safe Supabase credentials.
2. Protect dashboard and scan routes so logged-out users are redirected to login.
3. Port a reusable dashboard shell from the prototype using shared dashboard components and plan-state helpers.
4. Preserve local/dashboard preference behavior with Supabase-aware persistence when available.
5. Add the Next env example and verify the production build.

## Success Criteria

- Login route sends a Supabase OTP request when configured.
- Dashboard and scan routes are no longer publicly reachable.
- Dashboard category structure is decomposed into reusable components.
- The Next build passes after the port.
